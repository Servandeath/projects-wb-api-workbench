# Права доступа

В проекте используется ролевая модель доступа.

## Viewer

- видит список методов API;
- видит JSON-ответы;
- видит маскированные ключи.

## Tester

- вставляет временный ключ;
- выполняет тестовые запросы;
- смотрит JSON-ответы.

## Operator

- выполняет реальные API-запросы;
- сохраняет ответы в БД;
- импортирует локальные выгрузки.

## Admin

- добавляет и удаляет ключи;
- меняет настройки;
- управляет пользователями;
- может видеть полный ключ после подтверждения.
